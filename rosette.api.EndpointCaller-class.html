Class EndpointCaller

source code

EndpointCaller objects are invoked via their instance methods to obtain results from the Rosette server described by the API object from which they are created. Each EndpointCaller object communicates with a specific endpoint of the Rosette server, specified at its creation. Use the specific instance methods of the API object to create EndpointCaller objects bound to corresponding endpoints.

Use EndpointCaller.ping to ping, and EndpointCaller.info to retrieve server info. For all other types of requests, use EndpointCaller.call, which accepts an argument specifying the data to be processed and certain metadata.

The results of all operations are returned as python dictionaries, whose keys and values correspond exactly to those of the corresponding JSON return value described in the Rosette web service documentation.

call(self, parameters)

source code 

Invokes the endpoint to which this EndpointCaller is bound. Passes data and metadata specified by parameters to the server endpoint to which this EndpointCaller object is bound. For all endpoints except name-translation and name-similarity, it must be a DocumentParameters object or a string; for name-translation, it must be an NameTranslationParameters object; for name-similarity, it must be an NameSimilarityParameters object. For relationships, it may be an L(DocumentParameters).

In all cases, the result is returned as a python dictionary conforming to the JSON object described in the endpoint's entry in the Rosette web service documentation.

Parameters:
  • parameters (For name-translation, NameTranslationParameters, otherwise DocumentParameters or str) - An object specifying the data, and possible metadata, to be processed by the endpoint. See the details for those object types.
Returns:
A python dictionary expressing the result of the invocation.
